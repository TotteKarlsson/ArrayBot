Detailed Description

This section details the Structures and Functions relavent to the Filter Flipper
For an example of how to connect to the device and perform simple operations use the following links:

  • Example of using the Thorlabs.MotionControl.FilterFlipper.DLL from a C or C++ project.
    This requires the DLL to be dynamical linked.
  • Example of using the Thorlabs.MotionControl.FilterFlipper.DLL from a C# project
    This uses Marshalling to load and access the C DLL.

The Thorlabs.MotionControl.FilterFlipper.DLL requires the following DLLs

  • Thorlabs.MotionControl.DeviceManager.

Enumeration Type Documentation

enum FF_IOModes : WORD

FilterFlipper IO operations.

Enumerator
FF_ToggleOnPositiveEdge 

Input Mode - FilterFlipper toggles when signalled (See FF_SignalModes).

FF_SetPositionOnPositiveEdge 

Input Mode - FilterFlipper goes to position when signalled (See FF_SignalModes)

FF_OutputHighAtSetPosition 

Output Mode - Output signal set to match position, where 2 = Hi, 1 = Lo.

FF_OutputHighWhemMoving 

Output Mode - Output signal high when FilterFlipper is moving.

enum FF_SignalModes : WORD

FilterFlipper IO operations.

Enumerator
FF_InputButton 

Signal is raised when Button Pressed i.e. Hi to Lo. Action taken is defined by FF_IOModes.

FF_InputLogic 

Signal is raised on on rising edge Lo to Hi. Action taken is defined by FF_IOModes.

FF_InputSwap 

when set will swap Position 1 and 2.

FF_OutputLevel 

Output is set to either be hi when flipper at Position 2 or flipper is moving (see FF_IOModes).

FF_OutputPulse 

Output is set to pulse when flipper operates, either when flipper reaches Position or flipper starts moving (see FF_IOModes).

FF_OutputSwap 

when set will swap output logic levels.
